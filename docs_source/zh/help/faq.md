# 常见问题

## 基本使用

### 什么是 ZenMux？

ZenMux 是一个企业级 AI 模型聚合平台，提供数百种 AI 模型的统一 API 访问。我们支持包括 OpenAI GPT、Claude、Gemini、Qwen 等主流模型，让您通过一个 API 接口就能使用所有模型。

### ZenMux 有哪些优势？

::: info 核心优势

- **统一接口**：通过一个 API 访问数百种 AI 模型
- **智能路由**：自动选择最佳模型和提供商
- **故障转移**：自动切换到可用的服务提供商
- **成本优化**：动态选择性价比最高的模型
- **高可用性**：99.9% 服务可用性保证
- **全球部署**：多区域部署，低延迟访问
  :::

### 支持哪些 AI 模型？

我们支持超过 200 种 AI 模型，涵盖以下类型：

- **文本生成**：GPT-4、Claude 3、Gemini Pro、Qwen、ChatGLM 等
- **图像生成**：DALL-E、Midjourney、Stable Diffusion 等
- **语音识别**：Whisper、Azure Speech 等
- **图像识别**：GPT-4V、Claude 3 Vision 等

::: tip 快速上手
查看[完整模型列表](/zh/models/)了解所有支持的模型及其特性
:::

### 如何开始使用？

::: details 开始使用的步骤

1. 访问 [ZenMux 控制台](https://console.zenmux.ai) 注册账户
2. 获取 API Key
3. 查看[快速开始指南](/zh/guide/quickstart)
4. 开始发送 API 请求
   :::

## 账户与计费

### 如何获取 API Key？

按照以下步骤获取您的 API Key：

1. 登录 [ZenMux 控制台](https://console.zenmux.ai)
2. 在左侧菜单中点击「API Keys」
3. 点击「创建新的 API Key」
4. 为 API Key 设置名称和权限

## 技术问题

### API 请求格式是什么？

ZenMux 完全兼容 OpenAI API 格式，您只需要替换 API 端点即可：

::: code-group

```bash [curl]
# 原 OpenAI API
curl https://api.openai.com/v1/chat/completions

# ZenMux API
curl https://api.zenmux.ai/v1/chat/completions  # [!code highlight]
```

```python [Python]
# 只需修改 base_url
from openai import OpenAI

client = OpenAI(
    base_url="https://api.zenmux.ai/v1",  # [!code highlight]
    api_key="your-zenmux-api-key"  # [!code highlight]
)
```

:::

详细的 API 文档请查看[基本使用指南](/zh/guide/basic)。

### 如何设置智能路由？

在请求中添加 `zenmux-route` 参数实现智能路由：

::: code-group

```json [成本优先]
{
  "model": "gpt-4",
  "messages": [...],
  "zenmux-route": {
    "strategy": "cost",  // [!code highlight]
    "fallback": ["claude-3", "gemini-pro"]
  }
}
```

```json [速度优先]
{
  "model": "gpt-4",
  "messages": [...],
  "zenmux-route": {
    "strategy": "latency",  // [!code highlight]
    "fallback": ["gpt-3.5-turbo", "claude-instant"]
  }
}
```

:::

详细配置请参考[参数映射指南](/zh/guide/parameter-mapping)。

### 支持流式响应吗？

完全支持流式响应，设置 `stream: true` 即可：

```json
{
  "model": "gpt-4",
  "messages": [...],
  "stream": true  // [!code highlight]
}
```

### 如何处理错误响应？

ZenMux 返回标准的 HTTP 状态码和详细错误信息：

| 状态码 | 错误类型           | 处理建议               |
| ------ | ------------------ | ---------------------- |
| `400`  | 请求参数错误       | 检查请求参数格式       |
| `401`  | API Key 无效       | 验证 API Key 是否正确  |
| `403`  | 权限不足或余额不足 | 检查账户余额和权限设置 |
| `429`  | 请求频率过高       | 实施请求频率控制       |
| `500`  | 服务器内部错误     | 稍后重试或联系技术支持 |

### 有请求频率限制吗？

不同模型有相应的频率限制：

::: info 频率限制说明

- **GPT 模型**：每分钟 60 次请求
- **Claude 模型**：每分钟 100 次请求
- **图像模型**：每分钟 20 次请求
  :::

::: tip 提升限额
如需更高频率限制，请联系我们的销售团队申请定制方案
:::

## 安全与隐私

### 数据安全如何保障？

我们采用多重安全措施确保您的数据安全：

::: info 安全措施

- **传输安全**：所有 API 请求均通过 HTTPS 加密
- **存储安全**：不存储您的请求内容和响应数据
- **访问控制**：API Key 支持细粒度权限控制
- **合规认证**：通过 SOC 2 和 ISO 27001 认证
  :::

### 请求日志会被保存吗？

我们严格遵循数据隐私原则：

- **元数据**：保存请求的元数据（时间、模型、token 数量等）用于计费和监控
- **内容数据**：不保存请求的具体内容（prompt、response 等）
- **保存期限**：元数据保存 90 天后自动删除

### 支持私有部署吗？

我们提供灵活的企业级部署方案：

::: details 部署选项

- **本地部署**：在您的私有云环境中部署

- **混合部署**：结合公有云和私有云的混合方案
  :::

详情请联系我们的销售团队获取定制方案。

## 故障排除

### API 请求失败怎么办？

按照以下步骤进行故障排除：

1. **检查 API Key**：确认 API Key 是否正确且有效
2. **检查余额**：确认账户余额是否充足
3. **检查参数**：确认请求参数格式是否正确
4. **查看错误码**：根据返回的错误码进行相应处理
5. **联系支持**：如果问题持续存在，请联系技术支持

### 响应速度慢怎么办？

优化响应速度的建议：

::: tip 性能优化建议

1. **选择合适的模型**：某些模型处理速度更快
2. **使用智能路由**：让系统自动选择最快的提供商
3. **减少请求内容**：过长的 prompt 会增加处理时间
4. **检查网络**：确认网络连接是否稳定
   :::
