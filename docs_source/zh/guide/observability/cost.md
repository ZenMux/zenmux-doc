# 成本分析

ZenMux 提供了强大的成本分析功能，帮助您实时监控和分析 API 使用所产生的费用。通过成本分析页面，您可以查看不同维度下的支出明细，包括按模型、API Key、服务提供商等分类的成本数据，从而更好地优化应用使用策略，控制开支。

![cost](https://cdn.marmot-cloud.com/storage/zenmux/2026/01/16/kOgy6cO/8BB449C3-1DE9-421F-AFAF-DF102007A83C.png)

> **提示**：当前页面显示的数据不包含订阅计划中的使用数据，仅统计即用即付（Pay As You Go）的部分。

## 一、成本概览

在页面顶部，您可以看到一组关键成本指标的统计卡片，用于快速了解当前时间段内的整体开销情况。

### 可选过滤条件
您可以通过以下下拉选项自定义查询范围：
- **时间粒度**：支持按“Month”、“Day”、“Hour”等维度查看。
- **时间范围**：默认显示 UTC 时间下的最近周期，可手动选择具体日期或区间。
- **API Keys**：筛选特定的 API 密钥产生的成本。
- **模型选择**：可选择全部模型或指定某个模型（如 `google/gemini-3-pro-preview`）进行分析。

### 指标说明

| 指标 | 含义 |
|------|------|
| **Total Cost** | 总计花费金额，涵盖所有输入、输出及其他相关费用。 |
| **Input Cost** | 输入 Token 所产生的费用。 |
| **Output Cost** | 输出 Token 所产生的费用。 |
| **Other Cost** | 其他非 Token 相关的额外费用（例如调用延迟补偿、系统资源消耗等）。 |
| **Average Cost Per ...** | 平均每条请求/每单位 Token 的成本（根据上下文动态展示）。 |


---

## 二、成本按模型分布

该区域以图表形式展示各模型的消费占比，帮助识别哪些模型是主要的成本来源。

- 支持横向对比多个模型的使用成本。
- 点击某一项可跳转至该模型的详细成本报告。
- 若无数据，则显示为空白状态。

> ✅ 建议：定期检查此部分，及时调整高成本低效模型的使用频率。

---

## 三、成本按类型分布

此模块展示了成本在不同类型的分配情况，例如：
- 按 **模型类型**（如 LLM、Embedding、Vision）
- 按 **服务提供商**（如 OpenAI、Google Gemini、Anthropic）
- 按 **请求用途**（如聊天、推理、图像生成）

> 📊 图表会自动加载并可视化各类别的成本构成，便于直观理解支出结构。

---

## 四、成本按 API Key 分布

该部分列出了每个 API Key 对应的消费金额，方便团队管理多账号场景下的成本归属。

- 每个 API Key 显示其对应的总成本及使用频率。
- 支持导出为 CSV 或复制链接分享。
- 可结合“Usage”页面进一步分析具体调用行为。

> 🔐 安全提示：请勿泄露 API Key，建议为不同项目分配独立密钥以便追踪与审计。

---

## 五、提供者成本视图

切换到 “Provider Cost” 标签页后，可查看各个云服务提供商的具体成本明细。

1. **Cost by Provider**  
   - 展示不同供应商（如 Google、OpenAI）的总支出。  
   - 支持排序与筛选，便于比较性价比。

2. **Provider Details**  
   - 列出该提供商下所有模型的调用详情，包括：  
     - 模型名称  
     - 调用次数  
     - 输入/输出 Token 数量  
     - 单次调用成本  
     - 总费用  

> 💡 示例：若发现某提供商的模型单价偏高，可考虑迁移至更具性价比的替代方案。
