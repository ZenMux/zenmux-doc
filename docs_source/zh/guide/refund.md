# 退款

如果你对当前订阅或按量付费充值不满意,可以在符合条件的情况下申请退款。本文将详细介绍退款资格、计算方式和操作流程。

## 退款资格

ZenMux 的两种付费方式均支持退款,但需在规定时间内申请:

| 付费方式                        | 退款条件                                                                      |
| ------------------------------- | ----------------------------------------------------------------------------- |
| **订阅制**（Pro / Max / Ultra） | 订阅后 **24 小时内**,且等效 API 使用价值未超过订阅付款金额                    |
| **按量付费**（Pay-As-You-Go）   | 单笔充值订单支付后 **24 小时内**,退还该订单未使用的已付费额度（不含赠送额度） |

::: warning ⚠️ 退款时限
退款申请必须在**订阅或充值成功后 24 小时内**提交,逾期将无法发起退款。退款弹窗会显示剩余可退款时间（如 "退款窗口剩余 Xh Ym"）,请及时确认是否符合你的需求。
:::

### 不可退款的情况

以下情况无法申请退款:

| 情况                     | 说明                                  |
| ------------------------ | ------------------------------------- |
| **退款窗口过期**         | 订阅或充值超过 24 小时                |
| **已全额退款**           | 该订单已退还全部可退金额              |
| **退款处理中**           | 已有一笔退款正在处理,需等待完成       |
| **消耗超出可退金额**     | 支付后的 API 使用已消耗了所有可退额度 |
| **余额不足**（按量付费） | 充值钱包余额不足以处理退款            |
| **订单未完成**           | 订单尚未成功支付                      |
| **赠送订阅**             | 通过赠送（Gift）获得的订阅不可退款    |

## 退款计算方式

### 订阅制退款

退款金额按以下方式计算:

| 项目                   | 说明                              |
| ---------------------- | --------------------------------- |
| **已付总额**           | 当前订阅周期内已支付的全部费用    |
| **- 实际用量**         | 订阅期间已消耗的 API 调用等效价值 |
| **- 已退款**           | 之前已退款的金额（如有）          |
| **= 可退差额**         | 扣除用量和已退款后的剩余可退金额  |
| **- 平台手续费（5%）** | 平台收取的退款处理费用            |
| **= 最终退款金额**     | 实际退回到你支付方式的金额        |

即：**退款金额 =（已付总额 - 实际用量 - 已退款）×（1 - 5%）**

::: warning ⚠️ 无法退款的情况
如果你的等效 API 使用价值已**超过订阅付款金额**,则该订阅无法退款。
:::

### 按量付费退款

按量付费的退款以**单笔充值订单**为单位进行,每笔订单独立计算:

| 项目             | 说明                             |
| ---------------- | -------------------------------- |
| **订单金额**     | 该笔充值订单的支付金额           |
| **- 支付后消耗** | 支付该订单后已消耗的额度         |
| **- 已退金额**   | 该订单之前已退款的金额（如有）   |
| **= 退款金额**   | 已扣除 5% 手续费后实际退回的金额 |

::: tip 💡 赠送额度与邀请奖励说明

- 通过充值赠送（Discount）、平台赠送（Gift）、推荐奖励（Referral）等方式获得的免费额度**不可退款**,申请退款后这些额度将被收回。
- 若该笔订单关联了邀请奖励,退款后邀请人将**无法获得 $5 奖励**。
  :::

::: warning ⚠️ 在途请求说明
从付款到发起退款期间的在途 API 请求可能仍会结算,最终退款金额可能与退款预览中展示的金额不一致,以最终结算金额为准。
:::

## 退款流程

### 方式一：在线自助退款（订阅制）

**第 1 步：发起退款**

<img src="https://cdn.marmot-cloud.com/storage/zenmux/2026/03/20/fsjS4Sr/20260320154313.jpg" alt="发起退款" style="border-radius:5px; box-shadow: 0 4px 8px rgba(0,0,0,0.2);">

访问 [订阅管理页面](https://zenmux.ai/platform/subscription),在账单历史（Billing History）区域点击 **"退款"** 按钮。

> 注意：赠送订阅（Gift）的退款按钮会被禁用,提示"Gift subscriptions are not eligible for refunds"。

**第 2 步：确认退款详情**

<img src="https://cdn.marmot-cloud.com/storage/zenmux/2026/03/20/7AmamOi/20260320153822.jpg" alt="确认退款详情" style="border-radius:5px; box-shadow: 0 4px 8px rgba(0,0,0,0.2);">

系统将显示退款预览,包括当前计划、已付总额、实际用量、已退款金额、可退差额、手续费（含费率百分比）和最终退款金额。确认无误后点击 **"提交退款"**,需再次确认后方可提交。

**第 3 步：等待处理**

退款提交后,你可以在退款弹窗中查看处理状态:

| 状态                   | 说明                                       |
| ---------------------- | ------------------------------------------ |
| 🔵 处理中（Refunding） | 退款正在处理中,将在 **1-3 个工作日**内退回 |
| 🟢 成功（Refunded）    | 退款已完成,资金已退回到你的支付方式        |
| 🔴 失败（Failed）      | 退款无法处理,请联系客服获取帮助            |
| ⚪ 已取消（Canceled）  | 退款请求已被取消                           |

退款状态也会以标签的形式显示在账单历史记录的金额列中。

### 方式二：在线自助退款（按量付费）

**第 1 步：发起退款**

<img src="https://cdn.marmot-cloud.com/storage/zenmux/2026/05/22/EDazFTu/20260522115739.jpg" alt="发起退款" style="border-radius:5px; box-shadow: 0 4px 8px rgba(0,0,0,0.2);">

访问 [额度管理页面](https://zenmux.ai/platform/pay-as-you-go),在交易记录（Transactions）列表中,找到需要退款的 **Stripe** 或 **Antom** 类型充值订单,点击该行的 **"退款"** 按钮。

> 注意：只有 Stripe 和 Antom（支付宝）类型的充值订单支持在线退款,其他类型（如赠送、推荐奖励等）不可退款。超过 24 小时的订单退款按钮会变为灰色不可点击状态。

**第 2 步：确认退款详情**

<img src="https://cdn.marmot-cloud.com/storage/zenmux/2026/05/22/sTVTHos/20260522115812.jpg" alt="发起退款" style="border-radius:5px; box-shadow: 0 4px 8px rgba(0,0,0,0.2);">

系统将显示该笔订单的退款预览,包括订单金额、支付时间、支付后消耗、已退金额、最终退款金额和退款窗口剩余时间。确认无误后点击 **"提交退款"**,需二次确认后方可提交。

**第 3 步：等待处理**

退款提交后将自动处理,退款金额将在 **1-3 个工作日**内退回到原支付方式。退款金额会从你的充值钱包余额中扣除。

### 方式三：邮件申请退款（所有付费方式均可）

如果你需要通过邮件申请退款,请发送邮件至 [support@zenmux.ai](mailto:support@zenmux.ai),并提供以下信息:

- 账户邮箱 / 账户 ID
- 购买或订阅的日期和时间（格式：YYYY-MM-DD HH:mm）
- 购买或订阅的金额
- 当前未使用的额度余额
- 退款原因

我们将在收到邮件后 **7 个工作日内**审核你的退款申请,符合条件的退款将按原支付方式、原支付币种退回到你的原支付账户。

::: details ⏱️ 退款到账时间
由于不同支付机构的处理速度不同,退款实际到账可能需要额外 **3-10 个银行工作日**。
:::

## 支持的支付方式

退款将原路退回到你的原支付渠道:

| 支付渠道                    | 适用范围          |
| --------------------------- | ----------------- |
| **Stripe**（信用卡/借记卡） | 订阅制 + 按量付费 |
| **Antom**（支付宝等）       | 按量付费          |

## 退款后的影响

退款成功后,请注意以下变化:

| 付费方式     | 退款后影响                                                                                                  |
| ------------ | ----------------------------------------------------------------------------------------------------------- |
| **订阅制**   | 订阅立即降级为**免费计划（Free）**                                                                          |
| **按量付费** | 退款金额从充值钱包余额中扣减,关联的赠送额度将被**收回**;若该订单关联了邀请奖励,邀请人的 $5 奖励将被**撤销** |

- 退款资金将在 **1-3 个工作日**内退回到原支付方式
- 待处理的 API 调用可能会影响最终退款金额
- 因汇率波动产生的差异由用户自行承担

## 常见问题

### 退款后还能再次订阅吗？

可以。退款后你的账户会降级为免费计划,随时可以重新选择订阅方案。

### 赠送的额度可以退款吗？

不可以。通过充值赠送、平台赠送、推荐奖励等方式获得的免费额度不可退款,申请退款后这些额度将被收回。

### 赠送的订阅可以退款吗？

不可以。通过 Gift 方式获得的赠送订阅不支持退款,退款按钮会被禁用。

### 退款多久到账？

在线自助退款通常 **1-3 个工作日**到账;通过邮件申请的退款,审核周期为 **7 个工作日**,到账还需额外 **3-10 个银行工作日**。

### 已使用超过订阅金额,还能退款吗？

不能。如果你的 API 使用等效价值已超过所付订阅费用,则无法退款。

### 按量付费的退款是按订单还是按账户？

按**单笔充值订单**退款。你需要在交易记录中找到具体的充值订单并发起退款,每笔订单独立计算可退金额。

### 退款后关联的邀请奖励会怎样？

如果退款的充值订单关联了邀请奖励,退款后邀请人将无法获得该笔订单对应的 $5 推荐奖励。

### 按量付费退款提示"余额不足"怎么办？

退款金额会从你的充值钱包余额中扣除。如果当前余额不足以覆盖退款操作,系统将拒绝退款。请确保钱包中有足够的充值余额。

::: tip 💡 需要帮助？
如果你对退款计算有疑问或需要人工协助,请发送邮件至 [support@zenmux.ai](mailto:support@zenmux.ai)。
:::
