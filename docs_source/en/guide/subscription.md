# Subscription Plans

ZenMux Builder Plan subscriptions provide individual developers with a fixed monthly fee and a predictable way to call AI models‚Äîso you can focus on building and creating without worrying about per-request API costs.

## What Are Flows?

A **Flow** is the metering unit used in ZenMux subscriptions. It combines token usage and request count. For an intuitive reference:

- **1 Flow ‚âà 1 request in a Claude Sonnet 4.5 coding scenario, or 10 requests in a Claude Haiku 4.5 coding scenario**, where a typical coding scenario is **30,000 input tokens + 1,000 output tokens**.

This approach ensures fair billing across different models and usage patterns, giving you clearer visibility into how your subscription quota is consumed.

### Rate Limits

- **Quota window:** Refreshes in a rolling 5-hour window
- **Weekly limit:** Resets in a rolling weekly window

### Applicable Scenarios

‚úÖ **Allowed:**

- Personal development and learning
- Vibe coding and rapid prototyping
- Technical exploration and experiments
- Personal projects and non-commercial applications

‚ùå **Not allowed:**

- Production environments that are already live

- End-user‚Äìfacing applications
- Abusive behavior such as pooled multi-account rotation

## Using Subscriptions in Studio Chat

In addition to using your API key in developer tools, you can also chat directly on the web via **[ZenMux Studio Chat](https://zenmux.ai/chat)** using your subscription quota.

### Switch Billing Mode

On the Studio Chat page, you can flexibly choose whether each conversation uses subscription quota or Pay As You Go balance.

In the conversation settings, you‚Äôll see **Billing Mode**:

- **Subscription** - Uses subscription quota (Flows) and does not consume your account balance
- **Pay As You Go** - Charges your account balance based on actual usage

::: tip üí° Flexible switching
You can use different billing modes in different conversations. For example:

- Daily development, learning, prototyping ‚Üí Subscription
- Production testing, commercial validation ‚Üí Pay As You Go

You can switch at any time. The two modes do not affect each other, so you can always choose the best option for each scenario.
:::

## FAQ

### What‚Äôs the difference between subscriptions and Pay As You Go?

| Feature | Subscription (Builder Plan) | Pay-As-You-Go |
|-----|---------------------|----------------------|
| **Billing model** | Fixed monthly fee | Pay for actual usage |
| **Best for** | Personal development, learning | Production, commercial apps |
| **Cost predictability** | ‚úÖ High (fixed monthly fee) | ‚ö†Ô∏è Medium (varies by usage) |
| **Rate limits** | 10‚Äì15 RPM | Higher, customizable |
| **SLA** | Standard | Higher |
| **Value leverage** | ‚úÖ 5‚Äì10√ó | Standard API pricing |

### When does quota reset?

- **5-hour window:** Uses a rolling window mechanism, metered from when you start sending requests, and resets once every 5 hours
- **Weekly limit:** Metered from when you start sending requests, and resets every 7 days

You can view remaining quota and reset times in real time on the [Subscription Management page](https://zenmux.ai/platform/subscription).

### Can I use multiple plans at the same time?

No. One account can only have one active subscription plan at a time. If you need a higher quota, upgrade to Max or Ultra.
