## Parameter Reference

### Basic Parameters
- temperature: Controls the randomness of the output (0.0-2.0)
- max_tokens: Maximum number of tokens to generate
- max_completion_tokens: Maximum number of tokens for the completion part
- top_p: Nucleus sampling parameter; controls the probability threshold for token selection
- stream: Whether to enable streaming output
- stream_options: Additional options for streaming output
- stop: Stop sequences for terminating generation

### Advanced Parameters
- n: Number of responses to generate
- presence_penalty: Presence penalty to reduce repetition
- frequency_penalty: Frequency penalty to reduce overuse of frequent tokens
- logit_bias: Bias settings for specific tokens
- user: User identifier
- seed: Random seed to ensure reproducibility

### Function Parameters
- functions: Function call definitions (legacy)
- function_call: Function call control (legacy)
- tools: Tool call definitions
- tool_choice: Tool selection policy
- response_format: Response format control
- logprobs: Whether to return log probabilities
- top_logprobs: Number of top log probabilities to return
- extra_headers: Additional HTTP request headers

## Usage Recommendations

1. Parameter compatibility check: Before using a specific parameter, verify that the target model supports it.
2. Graceful degradation: ZenMux automatically ignores unsupported parameters to ensure the request executes successfully.
3. Best practices: Prefer widely supported basic parameters; use advanced features as needed.

> Note: Parameters marked as "Model-dependent" in the table may vary by specific model. We recommend testing and validation before use.
